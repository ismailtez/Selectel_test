Ниже представлен манифест Dockerfile, который содержит несколько
недочетов.
В качестве ответа укажите на текущие проблемы, оставив комментарий
по каждой инструкции, и опишите, как их можно исправить, оптимизировав
при этом использование слоев итогового образа.

FROM ubuntu:latest
MAINTAINER MyCompany
COPY . /var/www/html
RUN apt-get update -y
RUN apt-get install -y nginx
CMD ["nginx", "-g", "daemon off;"]

1 - тег latest, не всегда подходит, поэтому корректней было бы использовать конкретную версию. Например - ubuntu:22.04
2 - метка MAINTAINER не актуальна, сейчас используется LABEL. Например LABEL maintainer="...."
3 - Использование COPY перед обновлением, если изменить что-то в коде, сборка замедлится при повторном создании.
4 - Разделение установки пакетов на 2 слоя
RUN apt-get update -y
RUN apt-get install -y nginx
5 - nginx запустится от root, что не является безопасным

ВАРИАНТ УЛУЧШЕНИЯ

FROM ubuntu:22.04

LABEL maintainer="...."

RUN apt-get update -y && \
    DEBIAN_FRONTEND=noninteractive apt-get install -y --no-install-recommends nginx && \
    rm -rf /var/lib/apt/lists/*

COPY . /var/www/html

RUN useradd -m -u 1000 appuser && chown -R appuser:appuser /var/www/html
USER test

CMD ["nginx", "-g", "daemon off;"]
